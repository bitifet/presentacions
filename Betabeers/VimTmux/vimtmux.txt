    * Què és?

    * Terminal VS entorn gràfic.

    * Filosofia *Unix.
       - Tot és un fitxer (stdin, stdout, stderr), /dev, /proc, pipes...

    * Tipus de terminal
       - tty(s)
       - xterm / gterm / konsole...
       - guake / yakuake / tilda...
       - screen / tmux
       - "cygwin"

    * Unicode, codis ANSI...

    * Editors
       - vi(m) / Emacs (Altres herbes: nano / pico / joe...)

    * Accés remot
       - ssh (port forwarding, ssh -X...) / scp (rsync, sshfs, fish...)
       - "putty" / "winscp"

   * Intèrprets:
     - bash
     - altres (csh, zsh, ipython...)

   * Comandes bàsiques (xxxx):
     - echo, cat, touch, ln (-s), ls, rm, mkdir, cd, rmdir...
     - find, cat, grep, head, tail, sed (awk, perl, php, python, node...)
     - man (RTFM!!) (man bash / man man ...)

   * Altres comandes:
     - dict (dict RTFM)
     - ip / ifconfig / route (+...)
     - Específcs de cada distribució (apt/dpkg, rpm...)

   * Expressions regulars: ( http://www.regular-expressions.info/ )
     - POSIX (Standard, ampliament soportades).
     - Perl (Més potents, sintaxi més amigable IMHO (dict IMHO)).

   * Més: Google is your friend!! (Capitan Obvious)

Característiques:
=================

   * Administrador de finestres (ttys completament funcionals) en mode text.
   * Connexió / desconnexió remota sense tancar la sessió.
   * Divisió de les finestres en panells.
   * Altament personalitzable i més senzill que screen.
   * Tmuxinator.
   

tmux VS screen

   * Divisió de la finestra en panells.
      (screen només permetia dividir el viewport)

                                Vim (Vi-improved)










╻ ╻╻┏┳┓
┃┏┛┃┃┃┃
┗┛ ╹╹ ╹━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  * Origens: vi-improved (vim)

  * Sabors: vim / gvim / (futur?) neovim (http://bit.ly/1c1K4qe)

  * Inconvenients:
     - Corba d'aprenentatge ALTA (com totes les eines potents).
     - Vimscript.

  * Avantatges:
     - TOTS els de qualsevol altre editor:
       (Resaltat de sintaxi, snippets, folding, macros...)

     - Agilitat, escalabilitat, extensibilitat...
       (Tot coses que podem trobar també en alguns altres editors
       ...especialment Emacs)

     - Editor modal: (:h vim-modes) (normal, insert, visual...)
       (L'equivalent a dur la mà al ratolí és pitjar [Esc] SENSE llevar
       les mans del teclat)

     - Operacions simples combinables:
       Accións aplicables a objectes i/o moviments.
       «Why Atom Can't Replace Vim» http://medium.com/p/433852f4b4d1

  * Perquè Vim i no Emacs?
     - Emacs no és modal.
     - http://www.tiraecol.net/modules/comic/comic.php?content_id=3
       shotwell tiraecol-3.jpg
     - http://www.tiraecol.net/modules/comic/comic.php?content_id=10
       shotwell tiraecol-10.jpg
     - With Emacs you can't do :Sex ;-)


  * No Emacs?
     - http://orgmode.org/features.html
     - Però cerqui, compari... ;-)


  * The power of Text!!
    - Documents (markdown, rEstructuredText...)
    - Taules: csv
    - Integració amb la terminal:
        Execució:  :sh (shell) :! (qualsevol comanda + Filtrat!!)
        Captura:   :r (Fitxer)  - :r! (Stdout comanda)

  * Mapeig de tecles (:h map).

  * Comandes personalitzades (:h command).

    - Altres: Pathogen (vim package manager), LargeFile (també vim -b)...

  * Mode hexadecimal: :%!xxd / :%!xxd -r  (FAKE)
